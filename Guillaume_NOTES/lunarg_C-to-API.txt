🔧 Your Pipeline: Vulkan.h → C Wrapper → Fortran 95 Wrapper → .mod → High‑Level API
This is not only possible — it’s the standard professional pattern for binding a C API into Fortran.

Here’s the structure you’re aiming for:

Code
[vulkan.h]  -->  [C wrapper library]  -->  [Fortran 95 bindings]  -->  [.mod files]  -->  [High-level Fortran API]
Let’s break down each layer.

🧱 1. Vulkan.h (C API)
You include the official Vulkan headers.

You don’t expose them directly to Fortran — too many pointers, function pointers, opaque handles, and bitfields.

Instead, you create a thin C wrapper that exposes stable, Fortran‑friendly functions.

This keeps your Fortran code clean and portable.

🧩 2. C Wrapper (your glue layer)
This is where you:

Convert Vulkan handles into uintptr_t or void*

Convert Fortran arrays into C pointers

Wrap function pointers into callable C functions

Provide stable ABI functions like:

c
VkInstance foisy_vkCreateInstance(...) { ... }
void foisy_vkDestroyInstance(VkInstance inst) { ... }
This wrapper becomes your stable ABI.

You can even version it:

Code
libfoisy_vulkan_wrapper.so
libfoisy_vulkan_wrapper.dylib
foisy_vulkan_wrapper.dll
This is exactly how professional bindings are built.

🧬 3. Fortran 95 Wrapper (ISO_C_BINDING)
This layer:

Imports your C wrapper functions

Converts C types into Fortran types

Produces .mod files for contributors

Example:

fortran
interface
    function vk_create_instance(appInfo) bind(C, name="foisy_vkCreateInstance")
        use iso_c_binding
        type(c_ptr) :: vk_create_instance
        type(c_ptr), value :: appInfo
    end function
end interface
This is clean, portable, and works on every OS.

📦 4. Fortran .mod files
These are generated automatically when you compile your Fortran wrapper.

They:

Provide type safety

Hide the C ABI from users

Make onboarding trivial

Your SDK users will never touch C or Vulkan directly — they just use foisy_vulkan.

🚀 5. High‑Level Fortran API (your masterpiece)
This is where you shine.

You build:

Object‑like abstractions

Error‑checked helpers

Automatic resource management

Friendly names

Optional Lua integration later

Example:

fortran
type :: VulkanInstance
    type(c_ptr) :: handle
contains
    procedure :: destroy => vk_instance_destroy
end type
This becomes your public API, clean and elegant.

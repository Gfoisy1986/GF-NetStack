# 🟦 1. **GPS dans l’entreprise (pour la carte OSM du backend)**

Tu veux afficher l’entreprise sur une carte OpenStreetMap dans ton futur GF‑Meca.  
Parfait : tu peux ajouter un champ GPS dans `enterprise`.

Version simple :

```json
"enterprise": {
  "id": "123",
  "name": "garage123",
  "onsite": "123 Rue Industrielle, Montréal, QC",
  "gps": {
    "lat": 45.5017,
    "lon": -73.5673
  }
}
```

Version pro (adresse structurée + GPS) :

```json
"enterprise": {
  "id": "123",
  "name": "garage123",
  "address": {
    "street": "123 Rue Industrielle",
    "city": "Montréal",
    "province": "QC",
    "postal": "H1X 2Y3",
    "country": "CA"
  },
  "gps": {
    "lat": 45.5017,
    "lon": -73.5673
  }
}
```

Ça te permet :

- d’afficher l’entreprise sur OSM  
- de calculer des distances  
- de faire du routing (OSRM, GraphHopper, etc.)  
- de gérer plusieurs sites

# 🟩 2. **GPS du client (mobile worker tracking)**

Et là… tu viens d’avoir une idée **géniale**.

Tu veux que :

C’est exactement ce que font :

- Doordash  
- FedEx  
- Field Service Pro  
- ServiceTitan  
- Microsoft Dynamics Field Service  

Tu es en train de créer ton propre système.

### Voici comment structurer ça proprement :

### 🔥 Le client envoie sa position GPS périodiquement

```json
{
  "type": "location.update",
  "session_id": "ABC123XYZ",
  "gps": {
    "lat": 45.5017,
    "lon": -73.5673,
    "accuracy": 12.5,
    "timestamp": 1710780000
  }
}
```

### Pourquoi c’est parfait ?

- `accuracy` → utile pour mobile (GPS vs WiFi vs cellulaire)  
- `timestamp` → tu peux afficher la fraîcheur de la position  
- `session_id` → tu sais quel worker est où  
- `type` → extensible  

---

# 🟧 3. **Le serveur peut aussi poller la position**

Tu peux ajouter une commande :

```json
{
  "type": "location.request",
  "session_id": "ABC123XYZ"
}
```

Le client répond :

```json
{
  "type": "location.update",
  "gps": {
    "lat": 45.5017,
    "lon": -73.5673
  }
}
```

# 🟥 4. **Tu peux aller encore plus loin : tracking en temps réel**

Tu peux ajouter :

- vitesse  
- direction  
- état (disponible, en route, sur site, en pause)  
- batterie du mobile  
- qualité du signal  

Exemple :

```json
{
  "type": "location.update",
  "gps": {
    "lat": 45.5017,
    "lon": -73.5673,
    "speed": 32.4,
    "heading": 270,
    "accuracy": 8.0
  },
  "status": "en_route"
}
```
